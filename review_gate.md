# HITL Review Gates

## Overview
Each step in the 7-step business transformation workflow has Human-in-the-Loop (HITL) gates that require explicit approval before proceeding.

## Gate Definitions

### Step 1: Business Performance Dashboard
| Gate | Description | Status |
|------|-------------|--------|
| 1.1 | Review dashboard metrics (revenue splits, ops efficiency, competitor data) | PENDING |
| 1.2 | Approve performance baseline | PENDING |

### Step 2: Value Stream Analysis
| Gate | Description | Status |
|------|-------------|--------|
| 2.1 | Refine opportunity levers (Growth, Efficiency, Experience, Effectiveness) | PENDING |

### Step 3: SWOT to TOWS Action Engine
| Gate | Description | Status |
|------|-------------|--------|
| 3.1 | Validate SWOT entries derived from Steps 1 & 2 | PENDING |
| 3.2 | Approve TOWS action strategies | PENDING |

### Step 4: Triple-Layer Strategy & OKRs
| Gate | Description | Status |
|------|-------------|--------|
| 4.1 | Review Business, Digital, and Gen AI strategies | PENDING |
| 4.2 | Approve Strategic OKRs (Objectives + Key Results) | PENDING |

### Step 5: Digital Initiatives & RICE
| Gate | Description | Status |
|------|-------------|--------|
| 5.1 | User manual override of RICE values in UI | PENDING |
| 5.2 | Approve prioritized initiative list | PENDING |

### Step 6: Epic & Team Collaboration
| Gate | Description | Status |
|------|-------------|--------|
| 6.1 | Review team dependencies and technical blockers | PENDING |
| 6.2 | Approve Product OKRs | PENDING |

### Step 7: Feature Backlog & Roadmap
| Gate | Description | Status |
|------|-------------|--------|
| 7.1 | Review Delivery OKRs (Efficiency/Quality) | PENDING |
| 7.2 | Approve final roadmap with full traceability | PENDING |

## Approval Process
1. Review the data/output for the current step
2. Update the gate status to `APPROVED` or `REVISION_NEEDED`
3. Add review notes explaining the decision
4. Only proceed to the next step after all gates in the current step are APPROVED
